# Percona Platform — Project Guide

## What this repo is

Shared Go module (`github.com/percona/platform`) providing:
- **Protobuf API definitions** for Percona SaaS services (under `api/`)
- **Generated Go code** from those protos (under `gen/`, committed to the repo)
- **Shared Go libraries** consumed across all platform services (under `pkg/`)

The sibling repo `github.com/percona/saas` is a public mirror — `make saas` extracts a subset of this repo there, rewriting all import paths.

---

## Directory structure

### `api/` — Protobuf sources
| Path | Description |
|------|-------------|
| `api/telemetry/` | Telemetry reporter APIs (reporter, generic, pillars, events) |
| `api/utils/fsp/` | Filtering / Sorting / Pagination utility definitions |

### `gen/` — Generated code (do not edit by hand)
Mirrors `api/` structure. Regenerate with `make gen`.

### `pkg/` — Shared Go libraries
| Package | Description |
|---------|-------------|
| `pkg/app` | Common CLI flags and app configuration |
| `pkg/email` | AWS SES client |
| `pkg/fsp` | Generic FSP helpers for SQL queries |
| `pkg/fuzz` | Fuzzing helpers |
| `pkg/grpcutils` | gRPC utilities |
| `pkg/logger` | Structured Zap-based logger with request tracking |
| `pkg/ptls` | TLS utilities for servers |
| `pkg/rdata` | Request metadata |
| `pkg/servers` | Common gRPC/HTTP/debug server startup with interceptors |
| `pkg/sqlmetrics` | Prometheus metrics for SQL / reform ORM |
| `pkg/telemetry` | Telemetry helpers |
| `pkg/tracing` | Request tracing |

## Common commands

```bash
make init            # Build all dev tools into bin/ (run once after checkout)
make gen             # Regenerate all protobuf code (rm -rf gen, buf generate, format, breaking check)
make descriptors     # Update platform.bin used for breaking-change detection (run before make gen when removing protos)
make gen-code        # go generate + go install
make format          # gofumpt + goimports + buf format
make check           # go-consistent + golangci-lint
make test            # go test -race ./...
make test-cover      # Per-package coverage → cover.out
make test-crosscover # Cross-package coverage → crosscover.out
make swagger-ui      # Serve Swagger docs locally on :8081 (Docker required)
make saas            # Extract public subset to ../saas with import path rewrite
```

## Code generation workflow

1. Edit `.proto` files under `api/`
2. Run `make gen` — this deletes `gen/`, regenerates, formats, and checks for breaking changes against `platform.bin`
3. **When deleting protos:** run `make descriptors` first to update `platform.bin`, then `make gen`

Tools are built locally into `bin/` by `make init`. Never installed globally.

Generated code is **committed to the repo**.

---

## Key conventions

- Proto package naming: `percona.platform.<service>.v1` (always versioned)
- Generated packages go to `gen/<service>/` mirroring `api/<service>/`
- All dev tools live in `bin/` (built by `make init`, gitignored)
- `gen/` is wiped and fully regenerated by `make gen` — never edit by hand
- `//go:build` constraint (not the old `// +build` form) — Go 1.17+

## Dependency notes

- Module: `github.com/percona/platform`, Go 1.25.8
- gRPC: `google.golang.org/grpc` + `grpc-gateway/v2` for HTTP/JSON transcoding
- Logging: `go.uber.org/zap`
- Validation: `github.com/mwitkow/go-proto-validators`
- ORM: `gopkg.in/reform.v1` (PostgreSQL via `github.com/lib/pq`)

---

## CI

- **ci.yml**: on push to `main` and all PRs — runs `make gen`, `make gen-code`, cross-coverage tests, then `make format` + `make check` with reviewdog annotations on PRs
- **Verification step**: CI asserts no uncommitted source changes after gen (go.sum tidying is the only allowed diff)
